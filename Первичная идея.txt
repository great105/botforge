Общие слабости, которые я вижу
Все конкуренты болеют одними и теми же проблемами. У PuzzleBot, например, сами создатели признавали отсутствие "простой реализации конструктора по типу карты мыслей с перетаскиванием блоков" Vc.ru. SaleBot хвалят за функционал, но жалуются - "новичкам может показаться сложным, некоторые функции нужно ручками сидеть делать" A2is. Порог входа высокий у всех серьезных решений.
Где можно выиграть - план BotForge
1. AI-генерация бота по описанию на русском. Ни один конкурент не делает это нормально. Пользователь пишет: "Бот для записи в барбершоп с оплатой через ЮКассу", а конструктор генерирует готовую схему из блоков, которую потом можно руками дотюнить. Это убийственная фича для лид-магнита.
2. Превью бота прямо в интерфейсе. Ни у кого нет встроенного эмулятора Telegram. Везде надо деплоить и тестировать в реальном мессенджере. А у нас - окошко "телефон" справа от холста, где бот работает как в настоящем ТГ. Клик на "Тест" - и ты сразу видишь диалог.
3. Шаблоны под конкретные ниши. Не абстрактный "шаблон воронки", а "Барбершоп с записью", "Доставка еды", "Онлайн-школа с оплатой курса", "Техподдержка с AI". Готовые, работающие, бери и меняй тексты.
4. Бесплатный тариф без ограничений по функциям. У SaleBot бесплатно только 50 сообщений в сутки. У PuzzleBot - 150 подписчиков. А мы даем все блоки бесплатно, лимитируем только количество ботов (1 бот, до 100 подписчиков). Хочешь больше - плати. Простая модель.
5. Экспорт в код (aiogram 3). Это наш секретный ход как студии разработки. Пользователь собрал бота в конструкторе, потыкал - работает. Но ему нужна кастомная логика, которую в блоках не сделать. Он нажимает "Экспорт в Python" и получает чистый aiogram 3 код. Дальше или сам допиливает, или приходит к нам в студию. Конкуренты так не делают - они vendor-lock.
6. Холст как у Figma, а не таблица. Большинство конструкторов выглядят как Excel с кнопками. Наш прототип уже на правильном пути - визуальный граф с Безье-кривыми, зум, пан, цветовое кодирование.

Фронтенд (конструктор)
React + React Flow - это ключевое решение. React Flow - это библиотека специально для node-based редакторов (как Figma, n8n, Retool). Из коробки дает drag-and-drop ноды, кривые Безье между ними, зум/пан, мини-карту, группировку. Мы в прототипе написали все руками, но в продакшене React Flow сэкономит месяцы. Плюс Zustand для стейта вместо Redux - легче, быстрее, меньше бойлерплейта.
Стилизация - Tailwind + shadcn/ui. Быстро, консистентно, темная тема из коробки.
Бэкенд
FastAPI - однозначно. Он у тебя уже в руках, плюс для этого проекта идеально ложится: REST API для CRUD схем ботов, WebSocket для live-превью в эмуляторе, Pydantic для валидации JSON-схем блоков. Async нативно, что критично когда 100+ ботов крутятся одновременно.
Движок исполнения ботов
aiogram 3 + собственный интерпретатор графа. Схема из конструктора сохраняется как JSON-граф (ноды + edges). Движок на Python берет этот JSON и при каждом апдейте от Telegram шагает по графу: текущий узел -> обработка -> переход к следующему. Состояние каждого юзера (в каком узле он сейчас, его переменные) хранится в Redis.
Тут важный архитектурный выбор - один процесс aiogram на все боты через кастомный диспатчер, или отдельный процесс на каждого бота. Для старта - один процесс с мультиплексором по bot_id. Когда вырастешь до 500+ ботов - перейдешь на worker pool.
База данных
PostgreSQL для всего основного: пользователи, проекты, JSON-схемы ботов (jsonb колонка - удобно для запросов по структуре), подписчики ботов, логи, платежи. Redis для рантайма: состояние FSM каждого подписчика, кэш схем, очереди сообщений.
Инфра
Docker Compose на старте, потом при необходимости Kubernetes. Nginx как reverse proxy. Для вебхуков Telegram - один эндпоинт /webhook/{bot_token}, FastAPI маршрутизирует на нужного бота в движке.
AI-генератор
Claude API через Anthropic SDK. Пользователь описывает бота текстом, Claude генерирует JSON-схему в формате твоего графа. Промпт с примерами валидных схем, few-shot. Результат кидается прямо на холст React Flow.

React + React Flow + Tailwind | FastAPI + aiogram 3 + Pydantic | PostgreSQL + Redis | Docker
